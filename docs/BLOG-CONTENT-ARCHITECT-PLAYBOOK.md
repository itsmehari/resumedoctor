# Blog Content Architect Playbook

## Purpose

This playbook defines how ResumeDoctor blog content should be planned, written, and optimized for:

- global-English search reach
- clear reader trust
- higher blog-to-product conversion

## Voice system

- Clear, practical, confident.
- Tactical over motivational.
- Simple language first; avoid jargon unless needed.
- “Show me what to do next” style in every section.

## Core writing structure

Every post should follow this flow:

1. Hook: problem + outcome in first 120 words
2. Quick answer snapshot
3. Main framework (steps/comparison)
4. Practical examples
5. Common mistakes
6. FAQ (where relevant)
7. Internal next-step links
8. Primary + secondary CTA

## Conversion copy ladder

- Informational posts: soft CTA (examples/checklists)
- Comparison posts: medium CTA (templates/tools)
- Action posts: strong CTA (`/try`)

## Internal linking standard

Minimum 4 contextual in-body links per post:

- 2 links to same cluster
- 1 link to adjacent cluster
- 1 link to conversion target (`/try`, templates, or examples)

## Frontmatter standard

Required:

- `slug`
- `title`
- `description`
- `date`
- `updated`
- `author`
- `authorBio`
- `readTime`
- `tags`
- `coverImage`

Optional:

- `faq`
- `featured`

## Quality acceptance checklist

- Intro is specific and outcome-focused
- Headings match search intent
- Examples are concrete (not generic)
- Copy is skimmable (short paragraphs, bullets)
- CTA is clear and context-appropriate
- Metadata and tags are aligned with intent
